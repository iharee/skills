# Course Requirements Digest

This is a compact digest of `references/original-course-requirements.md`.

## Core Rules

- Data must be independently collected and current. Do not use stale financial or securities data; the cutoff should be as close to the present as the data source allows.
- The final bundle must include local data so the script can rerun directly after copying.
- References should preferably be journal papers or online papers from roughly the last one or two years; the closer to recent course expectations, the better.
- Use R as the required software.
- AI assistance is allowed.

## Recommended Workflow

1. Collect current data and save it locally for packaging.
2. If useful, run or adapt the instructor-provided R program as a baseline method or data-shaping reference.
3. Find recent literature and extract a usable method aligned with the current report theme.
4. Compare the literature-based or modified method with the baseline.
5. If the new method is worse, change the data, change the reference, or modify the method and compare again.

## Required Report Sections

1. 摘要
2. 使用方法与改进措施
3. 数据来源和采集时间
4. R程序分析结果，改进措施是否有效
5. 结论和展望
6. 参考文献

## Delivery Constraints

- Submit each classroom case report before class through QQ group.
- For the final course paper, select six case reports and compile them together.
- Each selected case is limited to one page.
- Bring the printed final paper to class.
- Package code and data for electronic submission.
- Before delivery, rerun the program once and confirm the written results match the actual output.
- Keep all required data in the current directory so the script can run directly after copying.
