# 前沿博客追踪

每天抓取 Anthropic 和 OpenAI 的最新博客文章，翻译成中文，落到 `笔记/技术/前沿/` 目录下。

---

# 第一步：抓取文章列表

## 信息源

同时抓取以下四个信息源，获取最新文章列表：

**OpenAI（使用 RSS，可直接解析）**

fetch `https://openai.com/news/rss.xml`，解析出所有文章的标题、URL、发布日期。
过滤规则：只保留 URL 中包含 `/index/` 的条目（去除 `/academy/` 等教程类噪音）。

**Anthropic 官方新闻室（fetch 列表页）**

fetch `https://www.anthropic.com/news`，从页面文本中解析出文章标题、相对路径、发布日期。
文章完整 URL 格式：`https://www.anthropic.com` + 相对路径（如 `/news/claude-opus-4-7`）。

**Anthropic Claude Blog（fetch 列表页）**

fetch `https://claude.com/blog`，从页面文本中解析出文章标题、相对路径、发布日期。
文章完整 URL 格式：`https://claude.com` + 相对路径（如 `/blog/claude-code-desktop-redesign`）。

**Anthropic 对齐研究博客（使用 RSS）**

fetch `https://alignment.anthropic.com/rss.xml`，解析出所有文章标题、URL、发布日期。

## 筛选当日或近 3 日内的文章

取今天日期，筛选出发布日期在「今天」或「最近 3 天以内」的文章（因为任务是每日运行，±3 天的窗口可防止因节假日跳过导致漏掉的情况）。

如果四个信息源合计没有任何符合时间窗口的文章，**输出「今日无新文章，跳过」并结束任务**。

---

# 第二步：去重检查

扫描目录 `笔记/技术/前沿/` 下的所有已有 `.md` 文件名。

文件命名规则为 `YYYY-MM-DD-{slug}.md`，其中 slug 是文章 URL 的最后一段路径（如 `claude-opus-4-7`）。

对第一步筛出的每篇文章，检查其 slug 是否已在目录中存在对应文件。已存在则跳过该文章。

如果所有文章去重后均已处理过，**输出「今日文章均已处理，跳过」并结束任务**。

---

# 第三步：抓取正文并翻译

对每篇通过去重检查的文章，执行以下操作：

## 3.1 抓取正文

fetch 文章的完整 URL，提取正文主体内容（去除导航栏、页脚、广告等无关内容，只保留文章标题、发布日期、正文段落、小标题、代码块、图片）。

**图片 URL 处理规则**：原始页面中的图片可能以 Next.js 代理格式出现，形如 `/_next/image?url=https%3A%2F%2F…`，需要将 `url=` 参数的值做 URL decode，还原为真实的 CDN 图片地址（如 `https://www-cdn.anthropic.com/images/…`）。对于已经是直链的图片（如 `https://cdn.prod.website-files.com/…`）直接保留原 URL。

## 3.2 判断文章价值

判断文章是否值得收录，以下类型**直接跳过**（不写入文件）：
- 企业合作案例（如「XX 公司使用 ChatGPT 提升效率」）
- 纯商业公告（融资、团队变动、法律声明等）
- 学术论文摘要（没有实践意义的纯理论）

以下类型**优先收录**：
- 新模型/新产品发布
- 技术架构、设计理念类博文
- AI 工程实践、开发者最佳实践
- 安全、对齐研究
- 行业洞察与趋势分析

## 3.3 翻译与整理

将文章内容翻译成中文，要求：
- 忠实原文，不删减核心内容，不添加个人评论
- 专有名词保留英文原文并在首次出现时附中文翻译，如「Agents SDK（智能体软件开发工具包）」；后续重复出现只用英文即可
- 代码块、命令行、API 名称等保持原文，不翻译
- 使用中文全角标点符号
- 引号统一使用「」
- 文章结构（标题层级、段落）与原文保持一致
- **图片保留规则**：正文中的内容型图片（如架构图、截图、性能对比图、产品界面图）必须保留，使用 `![](图片真实URL)` 语法写入；纯装饰性图片（公司 Logo、引导按钮图标、背景纹理等）直接忽略，不写入文件

---

# 第四步：写入文件

## 4.1 确保目录存在

```bash
mkdir -p "笔记/技术/前沿"
```

## 4.2 文件命名

文件名格式：`YYYY-MM-DD-{slug}.md`

其中 `YYYY-MM-DD` 为文章的原始发布日期（不是今天抓取的日期），`slug` 为文章 URL 的最后一段路径。

示例：`2026-04-16-claude-opus-4-7.md`

## 4.3 文件内容格式

```
# {文章中文标题}

> 来源：{来源标识} | 原文：{原文 URL} | 发布日期：{YYYY-MM-DD}

{翻译后的正文}
```

来源标识对应规则：
- openai.com → `OpenAI Blog`
- anthropic.com/news → `Anthropic Newsroom`
- claude.com/blog → `Anthropic Claude Blog`
- alignment.anthropic.com → `Anthropic Alignment Science`

## 4.4 批量写入

对本次筛选出的所有文章（跳过低价值的）逐一完成翻译并写入对应文件。

---

# 第五步：完成报告

输出本次任务的执行摘要，格式如下：

```
## 前沿博客追踪 - {今日日期}

新增 {N} 篇文章：
- {文件名}：{中文标题}
- ...

跳过 {M} 篇（已存在 / 低价值）：
- ...
```

如果本次没有新增任何文章，简单说明原因即可。
